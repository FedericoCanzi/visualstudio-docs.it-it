# C26167
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

avviso C26167: Probabilmente si sta rilasciando un blocco \<lock\> non mantenuto nella funzione \<func\>.  
  
 L'avviso C26167 è simile all'avviso [C26117](../code-quality/c26117.md) eccetto che il livello di fiducia è inferiore.  Ad esempio, la funzione può contenere errori di annotazione.  
  
## Esempio  
 Il codice seguente genera C26167 nonché C26110.  
  
```cpp  
  
typedef struct _DATA {   
    CRITICAL_SECTION cs;   
} DATA;   
  
_Releases_lock_(p->cs) void Leave(DATA* p) {   
    LeaveCriticalSection(&p->cs); // OK   
}   
void ReleaseUnheldLock(DATA* p) { // Warning C26167  
    int i = 0;  
    Leave(p); // Warning C26110  
}  
  
```  
  
## Esempio  
 Il codice seguente correggerà questi avvisi.  
  
```cpp  
  
typedef struct _DATA {   
    CRITICAL_SECTION cs;   
} DATA;   
  
_Releases_lock_(p->cs) void Leave(DATA* p) {   
    LeaveCriticalSection( &p->cs );  
}   
  
void ReleaseUnheldLock( DATA* p ) {  
    EnterCriticalSection( &p->cs );  
    int i = 0;  
    Leave(p);  
}  
  
```
