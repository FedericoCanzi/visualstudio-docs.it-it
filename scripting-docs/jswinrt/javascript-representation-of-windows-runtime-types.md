# Rappresentazione JavaScript dei tipi di Windows Runtime
Nella tabella riportata di seguito viene descritto in che modo JavaScript rappresenta i tipi di Windows Runtime.  
  
> [!IMPORTANT]
>  Le funzionalità di Windows Runtime non sono disponibili per le applicazioni eseguite in Internet Explorer.  
  
## Tipi di Windows Runtime in JavaScript  
  
|Tipo di Windows Runtime|Rappresentazione JavaScript|Descrizione|  
|-----------------------------|---------------------------------|-----------------|  
|`UInt8`|`Number`|Un oggetto `Uint8` di Windows Runtime è un Integer a 8 bit non firmato, rappresentato come numero JavaScript.  Innanzi tutto un valore JavaScript viene convertito in un numero JavaScript e quindi viene seguito il processo `ToUint32`.  Se il valore del numero JavaScript non è compreso nell'intervallo di un Uint8, al risultato verrà applicato il modulo 2^8.|  
|`Int32`|`Number`|Un oggetto `Int32` di Windows Runtime è un Integer a 32 bit firmato, rappresentato come numero JavaScript.  Innanzi tutto un valore JavaScript viene convertito in un numero JavaScript e quindi viene seguito il processo `ToInt32`.  Se il valore del numero JavaScript non è compreso nell'intervallo di un `Int32`, al risultato verrà applicato il modulo 2^16.|  
|`Int64`|`Number`|Un oggetto `Int64` di Windows Runtime è un Integer a 64 bit firmato, rappresentato come numero standard se compreso nell'intervallo \[\-2^53, 2^53\].  In caso contrario, viene rappresentato come un valore numerico con un archivio di backup personalizzato che gestisce i 64 bit completi dei dati Integer.  Tutte le operazioni matematiche su questi valori numerici Int64 personalizzati fanno sì che il valore venga convertito in un numero standard compreso nell'intervallo \[\-2^53, 2^53\].  Se il valore non è compreso nell'intervallo, viene generato un errore del tipo.  Le eccezioni a questo processo di conversione sono le operazioni `==`, `===`, `SameValue` e `RelationalComparison`, che confrontano gli argomenti basati sui 64 bit completi quando i due argomenti passati rappresentano valori a 64 bit.  Se uno dei due argomenti è un numero JavaScript standard, queste operazioni convertono anche l'argomento in un numero JavaScript prima del confronto.  Un valore JavaScript viene assegnato direttamente se è anch'esso un valore Int64; in caso contrario, il risultato ottenuto applicando la conversione `ToInteger` al valore viene passato a Windows Runtime.  Se l'assegnazione del tipo ha esito negativo, viene restituita un'eccezione.|  
|`Uint64`|`Number`|Un oggetto `Uint64` di Windows Runtime è un Integer a 64 bit non firmato, rappresentato come numero standard se compreso nell'intervallo \[0, 2^53\].  In caso contrario, viene rappresentato come numero con un archivio di backup personalizzato che gestisce i 64 bit completi dei dati Integer non firmati.  Tutte le operazioni matematiche su questi valori Uint64 personalizzati fanno sì che il valore venga convertito in un numero standard compreso nell'intervallo \[0, 2^53\].  Se il valore non è compreso nell'intervallo, viene generato un errore del tipo.  Le eccezioni a questo processo di conversione sono le operazioni `==`, `===`, `SameValue`, and `RelationalComparison`, che confrontano gli argomenti basati sui 64 bit completi quando vengono passati due valori a 64 bit.  Se uno dei due argomenti è un numero JavaScript standard, queste operazioni convertono anche l'argomento in un numero JavaScript prima del confronto.  Un valore JavaScript viene assegnato direttamente se è anch'esso un valore Uint64; in caso contrario, il risultato ottenuto dall'applicazione della conversione `ToInteger` al valore, seguita dall'accettazione del modulo 2^52, viene passato a Windows Runtime.  Se la conversione del tipo ha esito negativo, viene restituita un'eccezione.|  
|`Single`|`Number`|Un oggetto `Single` di Windows Runtime è un numero a virgola mobile a 32 bit, rappresentato come numero JavaScript selezionando il valore a precisione doppia più vicino al valore a precisione singola.  Un valore JavaScript viene convertito in un numero JavaScript e quindi controllato nell'intervallo per verificare che possa essere rappresentato in un numero a virgola mobile IEEE 754 a 32 bit e a precisione singola.  Se la conversione o il controllo nell'intervallo ha esito negativo, viene restituito un errore di marshalling.|  
|`Double`|`Number`|Un Windows Runtime `Double` è un numero a virgola mobile a 64 bit.  `ToNumber` è utilizzato per convertire un `Double` di Windows Runtime in un numero JavaScript.  Se la conversione ha esito negativo, viene restituito un errore di marshalling.|  
|`Boolean`|`Boolean`|Un `Boolean` di Windows Runtime è un valore a 8 bit in cui zero è `false` e qualsiasi valore diverso da zero è `true`, rappresentato come `Boolean` JavaScript.  Un valore JavaScript in primo luogo viene convertito in un `Boolean` JavaScript da `ToBoolean`.  Questo significa che una funzione di Windows Runtime dichiarata come `void func(BOOL);` potrebbe essere scritta in JavaScript come `obj.func("test");`.  La funzione di Windows Runtime viene chiamata con il parametro `BOOL` passato come `TRUE`.  Se la conversione ha esito negativo, viene restituito un errore di marshalling.|  
|`Char16`|`String`|Un oggetto `Char16` di Windows Runtime è un carattere Unicode a 16 bit, rappresentato come stringa JavaScript contenente un singolo carattere.  Un valore JavaScript viene convertito in una stringa JavaScript da `ToString` e quindi controllato per verificare che la stringa contenga un solo carattere.  Il singolo carattere viene quindi passato come valore `Char16`.  Se la conversione o il controllo della lunghezza ha esito negativo, viene restituito un errore di marshalling.|  
|`String`|`String`|Un elemento `String` di Windows Runtime è una sequenza non modificabile di oggetti `Char16`.  Le stringhe sono rappresentate come oggetti `String` JavaScript.  Le stringhe di Windows Runtime non hanno valori diversi per le stringhe `null` e vuote \(""\).  I valori delle stringhe `null` e vuote vengono convertiti nella stringa vuota JavaScript.  Nota: quando l'elemento `null` di JavaScript viene passato a un parametro di Windows Runtime che prevede una stringa, produce un valore di stringa "null", non `null` o la stringa vuota \(""\).  Per le stringhe passate a Windows Runtime è necessario creare sempre un'istanza nella stringa vuota.|  
|`Enumeration`|`Object`|Un elemento `Enumeration` di Windows Runtime è un Integer a 32, firmato o non firmato, con un set associato di costanti denominate.  In JavaScript, un'enumerazione è rappresentata come oggetto contenente un campo di sola lettura per ogni valore denominato.  I valori di enumerazione vengono convertiti in numeri JavaScript definiti in precedenza per `Int32` e `UInt32`.  Quando un valore JavaScript viene convertito in un'enumerazione di Windows Runtime, viene convertito, troncato e controllato nell'intervallo come descritto in precedenza.  Tuttavia, il valore risultante non viene confrontato con i valori denominati definiti specificati nell'enumerazione.|  
|`Structure`|`Object`|Un elemento `Structure` di Windows Runtime è una raccolta eterogenea di campi dati denominati.  In JavaScript, una struttura viene rappresentata come oggetto JavaScript contenente una proprietà denominata per ogni campo della struttura.  Se la conversione di un valore della struttura qualsiasi ha esito negativo, viene restituito un errore di marshalling.  I campi nell'oggetto JavaScript che non hanno equivalenti nella struttura di Windows Runtime vengono ignorati.  Nota: non è possibile creare un'istanza di una struttura di Windows Runtime in JavaScript con la parola chiave `new`.|  
|`Array`|`Array`|Un elemento `Array` di Windows Runtime viene convertito in una matrice JavaScript.  Tuttavia, le matrici di Windows Runtime non sono ridimensionabili, pertanto alcune operazioni relative alle matrici JavaScript non sono possibili.  La proprietà interna \[\[Classe\]\] di una matrice convertita non viene impostata su "Array" perché questa operazione non è consentita dalla specifica ECMAScript 5.  Questo significa che `Array.isArray(v)` restituisce `false`.  Un valore JavaScript viene convertito in una matrice di Windows Runtime come segue: se il valore è `null` o `undefined`, viene convertito in `null` nativo.  Se la proprietà \[\[Class\]\] interna è "Array", viene eseguita la copia in una matrice nativa e viene passato un riferimento a questa matrice.  Se è una matrice convertita, come descritto in precedenza, viene passata la matrice nativa sottostante.  Nota: se si passa un valore di matrice JavaScript a un metodo di Windows Runtime, viene eseguita una copia completa della matrice.|  
|Delegato \(callback di funzione\)|`Function`|Un delegato di Windows Runtime è un riferimento a un singolo metodo.  Un delegato di Windows Runtime viene rappresentato in JavaScript come elemento `Function` JavaScript, che viene sempre chiamato sul thread appropriato.  Quando viene richiamato l'elemento `Function`, gli argomenti vengono convertiti nei tipi di parametro di Windows Runtime equivalenti.  Se il numero di argomenti JavaScript è inferiore a quello dei parametri `in` del delegato, la chiamata al delegato non riesce.  Se invece il numero di argomenti JavaScript è superiore al numero di parametri `in` nel delegato, gli argomenti JavaScript aggiuntivi vengono ignorati.  Dopo il richiamo del delegato, i parametri `out` vengono convertiti in tipi JavaScript e vengono restituiti.  Se un oggetto funzione JavaScript nativa viene convertito in un delegato di Windows Runtime, viene eseguito il wrapping in un delegato di Windows Runtime di tipo corrispondente.  Quando viene richiamato il delegato, i parametri `in` vengono convertiti in tipi JavaScript.  Dopo il richiamo del delegato, il valore restituito viene convertito nei parametri `out` del delegato.|  
|Interfaccia|`Object`|Le interfacce e i gruppi di interfaccia non sono rappresentati direttamente in JavaScript come oggetti.  Tuttavia, le interfacce sono rappresentate come tipi di parametro e restituiti dei metodi di Windows Runtime.  Un'istanza dell'interfaccia Windows Runtime viene convertita come segue:<br /><br /> 1.  Se esiste una classe di runtime valida, rappresentare l'oggetto come istanza di tale classe.<br />2.  Se non esiste una classe di runtime valida, rappresentare l'oggetto come istanza di una classe di runtime senza nome che implementa esattamente l'interfaccia solitamente implementata dall'istanza e le interfacce richieste.<br /><br /> Un valore JavaScript viene verificato per determinare se è un'istanza di una classe di runtime o di un'interfaccia.  Se lo è e il valore di Windows Runtime originale implementa l'interfaccia, l'oggetto viene passato a Windows Runtime.|  
|Classi di runtime|`Object`|Gli oggetti in Windows Runtime possono essere istanze delle classi runtime che implementano una o più interfacce.  Gli oggetti Windows Runtime sono rappresentati in JavaScript come oggetti.  L'unione di metodi, proprietà ed eventi definiti in tutte le interfacce implementate della classe rappresenta le proprietà denominate nel prototipo dell'oggetto JavaScript.  Gli utenti JavaScript possono accedere direttamente a qualsiasi membro della classe.  Gli oggetti Windows Runtime hanno un prototipo contenente tutti i membri di tutte le interfacce implementate della classe di runtime.|
